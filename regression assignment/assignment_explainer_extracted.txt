Missing Values — What was the problem and how did you fix it?
When you ran the code, 6 columns had 20 missing values each: CRIM, ZN, INDUS, CHAS, AGE, and
LSTAT. Missing values are a problem because machine learning models cannot handle blank entries
— they will crash or give wrong results.
The fix used was median imputation — replacing each missing value with the median of that column.
The median is the middle value when all numbers are sorted. It was chosen over the mean (average)
because housing data has outliers (a few extremely expensive or cheap houses), and the median is
not affected by those extremes the way the mean is.
In plain English: if the average number of rooms is 6.3 but one mansion has 20 rooms pulling the
average up to 7.1, the median stays at 6.3. That is more honest.
Standardization — Why did you scale the data?


===== PAGE 3 =====
Different columns have completely different scales. CRIM (crime rate) might be 0.006, while TAX (tax
rate) might be 296. If you feed these raw numbers into a model, the model will think TAX is 50,000
times more important than CRIM just because its numbers are bigger. That is wrong.
StandardScaler fixes this by converting every column so that its mean becomes 0 and its standard
deviation becomes 1. Now all features are on equal footing and the model can fairly compare them.

Outliers
40 houses were identified as outliers using the IQR method. IQR stands for Interquartile Range — the
range between the 25th and 75th percentile of prices. Any house priced more than 1.5x the IQR
above or below those boundaries is flagged as an outlier. Outliers can confuse the model by making it
try to fit unusual cases at the expense of typical ones.

PART D — Regression Modeling (25 Marks)
What are you doing here?
You are training two different machine learning models on the data and comparing their predictions.
Both models learn from the training set (404 houses) and are then tested on the test set (102 houses
they have never seen before). The 80/20 split means 80% of the data trains the model, 20% tests it.
Model 1: Linear Regression
Linear Regression assumes there is a straight-line relationship between the features and the target. It
finds the best straight line through the data. Think of it as drawing a line of best fit through a scatter
plot — the model learns the slope and position of that line from the training data, then uses it to
predict prices for new houses.
In plain terms: Linear Regression says 'each extra room adds X dollars to the price, each % of
lower-income residents removes Y dollars' and combines all those effects into one prediction.
Model 2: Decision Tree Regressor
A Decision Tree works completely differently. It learns a series of if-then rules from the data. For
example: IF rooms > 7 AND crime < 0.1 THEN price is around $35,000. It builds a tree of these rules
by splitting the data at each step to reduce prediction error. It does not assume any straight line — it
can capture complex, non-linear patterns.

The most important finding — Overfitting
This is the key insight of the entire assignment. On the test set alone, Decision Tree looks better:
lower RMSE (3.87 vs 4.19) and higher R² (0.7957 vs 0.7608). You might think Decision Tree wins.
But then you look at cross-validation. Cross-validation tests the model 5 different times on 5 different
splits of the data. This gives a much fairer picture of how the model performs generally.
Decision Tree CV R² Mean = 0.1417, Std = 0.8585. This is terrible. It means on average the model
explains only 14% of price variation across different data splits. The standard deviation of 0.86
means results jump wildly — sometimes it works okay, other times it completely fails. This is
OVERFITTING — the model memorized the training data instead of learning real patterns.
Linear Regression CV R² Mean = 0.6519, Std = 0.1620. Consistently explains 65% of price variation
across all splits. The low standard deviation (0.16) means it performs reliably every time. LINEAR
REGRESSION IS THE WINNER.
What is overfitting in plain English?
Imagine you are studying for an exam by memorizing every past paper answer word for word. You
score 100% on past papers. But when the real exam comes with slightly different questions, you fail
because you memorized answers instead of understanding the subject. That is what the Decision
Tree did — it memorized the training houses instead of learning the real relationship between
features and price.
Residual Plots (residual_plots.png)
A residual is the difference between what the model predicted and the actual price. A good model has
residuals scattered randomly around zero — no pattern. If you see a pattern (curve, funnel shape),
the model is missing something systematic. Linear Regression residuals should look more randomly

PART F — Presentation & Reflection (10 Marks)
Summary of key findings
1. The dataset has 506 houses, 14 features, and 6 columns had missing values filled with median.
2. House prices average $22,530 with a slight right skew — a few very expensive houses pull the
mean up.
3. The strongest predictors of price are LSTAT (poverty, -0.72) and RM (rooms, +0.70).
4. Feature engineering created LSTAT_PTRATIO which had -0.74 correlation — stronger than any
original feature.
5. Decision Tree appeared better on test set but was completely overfitting.
6. Linear Regression is the better model — consistent, reliable, generalizes to new data.
Limitations of the dataset
The Boston Housing Dataset was collected in the 1970s. Housing markets have changed enormously
since then — property values, crime patterns, school systems, and urban development are all
different. Any model trained on this data cannot reliably predict modern house prices.
With only 506 rows, the dataset is small by modern machine learning standards. Small datasets
increase the risk of overfitting and make cross-validation results less stable.
Suggested improvements
1. Use Ridge or Lasso Regression instead of plain Linear Regression. These add a penalty for overly
complex models, which helps prevent overfitting and often improves performance on new data.
2. Tune the Decision Tree using the max_depth parameter. Instead of letting the tree grow as deep
as it wants (which causes memorization), limit it to 4-6 levels. This forces it to learn general patterns
rather than specific training cases.
